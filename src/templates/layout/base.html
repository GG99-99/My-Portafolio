PROYECTOS

App Web de Turnos
Chat Web Grupal
Vista para Clientes
Esta es la vista de los clientes , donde pueden visualizar su nombre, turno asignado, el estado de su turno y el estado de la silla en la cual esperan.
Seccion de Informacion
Esta es un seccion de informacion en la misma vista de los clientes, donde se brindan datos de locaclizacion y contacto del negocio.
Panel de Administracion
Esta es la vista del panel administrador . Es donde el encargado de gestionar los turnos realizan los eventos que alteran la lista de espera.
Inicio de Sesion
Esta es la vista de iniciar sesion (login), para el encargado de gestionar los turnos. Cuenta con dos simples campos, nombre de usuario y contrasena.
Descripcion

Este proyecto de turnos se realizo con un efoque sencillo y minimalista para ayudar a peluqueros locales que tienen la intencion de profesionalizar el manejo de sus clientes, sin embargo este sistema se puede adaptar y extrapolar a muchos otros tipos de negocios donde el manego de turnos es clave para un sentimiento de conformidad y profesionalismo en los clientes

Detalles Tecnicos

Socket.io
Fue implementado para realizar cambios en tiempo real en la vista de clientes segun se generen desde el panel administrativo.

Eventos:

- Subir/Bajar Turno
- De/Des declinar Turno
- Abrir/Cerrar Silla
- Agregar Cliente
Json web tokens
Se utilizo para manejar y autorizar las sesiones de los administradores en el panel, tambien se ultizo como capa de seguridad para proteger quienes y desde donde se generan los eventos de socket.io.
db-local
Es la dependencia que se utilizo para el almazenaje de la informacion de los clientes y usuarios.
Vista General
Este es el apartado principal de la web, en el centro estan los mensajes que han sido agregados y en el lateral derecho las acciones que se pueden realizar
Formulario Crear Mensaje
Este overlay es la seccion donde el usuario ingresa la informacion para crear un Mensaje
Formulario para cambiar sesion
Este overlay es la seccion donde el usuario ingresa la sesion de mensajes a la que quiere ir. El funcionamiento de las sesiones sera explicado en la Descripcion
Vista de seleccion
Esta es la vista en el que podemos seleccionar los mensajes que seran destinados ha deshabilitarse ("elminarse"), para finalizar el proceso se solicitara una master-key
Concepto General

Este proyecto de mensajes, se realizo con el proposito de ayudar al club de poesia universitario del Intec, para realizar actividades colaboratibas que involucran la creacion y transmicion de poemas individuales.

Master-Key

La master-key es una especie de contrasena que solo conoce el lider del grupo, la cual permite ejecutar el evento de Deshabilitar un mensaje. El evento Deshabilitar ocurre de manera general con el io.emit de socket.io

Sesiones

Las sesiones son una funcionalidad que permite tener una especie de espacios de trabajo con distintos mensajes en el mismo dia. Solo se pueden visualizar los mensajes que sean del dia en curso y en la session que se encuentre en cliente. Las sesiones se manejan como eventos io individuales es decir de socket.emit de socket.io
